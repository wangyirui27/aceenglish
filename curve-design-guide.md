# 曲线路径设计指南：从数学模型到丝滑交互

在 UI 开发中，涉及复杂路径（如游戏关卡图、进度轨道）时，**“数学模型 > 手动硬编码”**。通过分离推进逻辑与装饰逻辑，可以实现既有节奏感又美观的路径布局。

## 1. 核心架构：推进与装饰分离
将路径生成解耦为两个维度：
*   **推进 (Y 轴)**：`y = i * unitHeight + offset`
    *   **作用**：保证各节点在逻辑推进上的物理距离一致，防止视觉拥挤。
*   **风景 (X 轴)**：`x = f(i)`
    *   **作用**：负责产生摆动感与路径模式的变化。

## 2. 数学模式库
通过不同的数学公式调制 X 轴，可获得截然不同的视觉风格：

| 模式 | 公式逻辑 | 视觉特点 | 推荐场景 |
| :--- | :--- | :--- | :--- |
| **Zigzag** | `i % 2 === 0 ? -A : A` | 硬核、机械、有折角 | 快节奏、强规则 |
| **Sine** | `sin(i * freq) * A` | 丝滑、自然、耐看 | 通用旅程感 |
| **Meander** | `sin(i*f1)*a1 + sin(i*f2)*a2` | 自然河流、随机感 | 探索/解谜关卡 |
| **Spiral** | `sin(t) * (1 - cos(t) * ratio)` | 非对称、戏剧化 | 章节高潮处 |

> **视觉甜区**：正弦波频率 `0.5 ~ 0.8` 是视觉最舒服的区间，太低则平淡，太高则产生锯齿感。

## 3. 贝塞尔连接的“平滑”秘诀
使用 **三次贝塞尔曲线 (Cubic Bézier)** 连接相邻节点，以避免折线感。
*   **控制点策略**：在两节点间的 Y 轴中点 `midY` 处设置两个控制点，并锁死其 X 轴坐标。
*   **公式示例**：
    `C prevX, midY, currentX, midY, currentX, currentY`
*   **原理**：强制路径在节点处切线趋于垂直，实现完美的 S 型平滑过渡。

## 4. 实现原则
1.  **振幅 (Amplitude) 克制**：建议设为容器宽度的 25%~35%，确保用户操作范围覆盖在屏幕安全区内。
2.  **GPU 友好**：呼吸、涟漪等节点动画使用 SVG 原生 `<animate>` 标签，避免引入 JS 状态循环导致的性能抖动。
3.  **系统化思维**：一旦确立了 Y 轴等分与贝塞尔对称控制点，无论如何调整频率或幅度，视觉基础逻辑都不会崩塌。
