## 核心颠覆认知

**媒介形态差异决定一切。** 9:16 不是 16:9 的裁剪版本，而是完全不同的认知范式：

| 维度 | 横屏 16:9 | 竖屏 9:16 |
|------|----------|----------|
| **媒介逻辑** | 空间叙事（还原物理空间） | 注意力锚定（情绪优先级） |
| **视觉基础** | 贴近人眼左右视野（水平视野是垂直2倍） | 贴近手机握持状态（拇指经济） |
| **提示词本质** | 场景描述型 | **优先级指令型** |
| **信息组织** | 环境→主体→动作→运镜 | **画幅→安全区→主体→情绪→运镜** |
| **用户行为** | 专注观看 | 碎片化浏览，3秒定生死 |

## 一、提示词结构根本差异

### ❌ 横屏思维（做竖屏就翻车）

```
雨夜的欧式街头，路灯昏黄，蒸汽从下水道溢出，
霸总走在街上，低角度仰拍，16:9 横屏
```

**问题**：环境优先，主体弱化，模型先按横屏渲染再裁剪 → 主体偏离/细节丢失

### ✅ 竖屏思维（优先级指令型）

```
9:16 竖屏，优先级1：霸总面部特写（占画面中间80%安全区）；
优先级2：低角度仰拍（相机位于女主腰部）；
优先级3：0.5秒快速推镜；
优先级4：背景完全虚化，仅保留电梯金属门冷色反光；
优先级5：冷色调，高对比度；
负向提示词：no 水平摇拍，no 背景杂乱，no 主体偏离安全区
```

**核心规则**：
1. **画幅比例必须前置**（最高权重）— AIGC 模型是"先确定画布，再填充内容"
2. **优先级排序**（1/2/3/4...）— 强制模型按优先级分配算力
3. **主体占比量化**（"占画面中间80%"）— 避免主体被背景淹没
4. **负向提示兜底** — 排除横屏思维的无关元素

## 二、安全区刚性规则（移动端必须）

> 核心认知：**安全区不是技巧，是刚性要求** — 手机顶部状态栏、底部评论按钮、右侧分享图标会遮挡画面，关键信息落在这些区域 = 观众错过核心情绪点。

### 平台安全区速查

| 平台 | 安全区范围 | 字幕安全区 |
|------|-----------|-----------|
| **国内**（抖音/快手/视频号） | 上下各避让 **20%**，中间 60% | 底部 20%-30% |
| **国际**（TikTok/Shorts） | 顶部 10% + 底部 10% 避让，中间 **80%** | 底部 10% |

⚠️ 抖音《竖屏内容创作规范》：不符合安全区要求会被扣 **10% 基础流量权重**

### 工具原生支持情况

| 工具 | 是否自动适配安全区 |
|------|------------------|
| Seedance 2.0 / Pika 3.0 / 可灵 AI 3.0 | ✅ 原生识别，自动锁定主体在中间区域 |
| Stable Video Diffusion v1.5 及以下 | ❌ 必须显式提示词约束，甚至像素级坐标 |

### 旧版工具的兜底提示词

```
9:16 竖屏，主体必须位于画面垂直方向 10%-90%、
水平方向 10%-90% 的安全区内，
顶部 10% 和底部 10% 不得出现任何关键元素
（人脸中心点坐标 450,500）
```

## 三、景别优先级：完全反转

### 横屏思维：全景 → 中景 → 特写（从环境到主体）

```
16:9 横屏，全景：雨夜街头；
中景：男主撑伞走向女主；
特写：女主的手搭上男主的伞柄
```

### 竖屏思维：特写 → 中近景 → 全景（从情绪到环境）

```
9:16 竖屏，开场特写：霸总面部（占中间80%安全区，0-3秒抓住注意力）；
中近景：展现霸总上半身动作；
全景：仅作为情绪反衬（如角色崩溃时的空荡房间）
```

### 行业数据

- 微短剧的**近景+特写占比通常超 70%**
- 情感类短剧甚至达到 **90%**

### 黄金 3 秒原则（竖屏专属）

**第一帧必须是大特写或特写**，直接锚定情绪注意力，**绝不用全景开场**。

## 四、竖屏专属 4 种镜头角度

横屏的"左右对打"思维完全不适配竖屏垂直画幅。以下 4 种是**竖屏原生角度**：

### 1. 低角度仰拍（竖屏放大压迫感）

**功能**：强化主体强势感（霸总/反派登场）

**竖屏优势**：垂直画幅让仰拍主体的轮廓**充满整个屏幕**，压迫感翻倍

```
9:16 竖屏，低角度仰拍（相机位于女主腰部位置），
霸总面部特写（占画面上半区80%安全区），
背景虚化，冷色调，压迫感拉满
```

### 2. 45° 侧俯拍（竖屏放大孤独感）

**功能**：营造孤独/脆弱感（女主崩溃/男主回忆）

**竖屏优势**：俯拍让主体渺小 + 竖屏垂直压缩 = 孤独感双重放大

```
9:16 竖屏，45° 侧俯拍，
女主蜷缩在床角（占画面中间80%安全区），
背景虚化，暖黄台灯打在发顶，孤独感拉满
```

### 3. ⭐ 垂直错位正反打（竖屏原创，对话神器）

**功能**：解决竖屏对话场景的构图难题

**原理**：横屏水平正反打会浪费 90% 纵向空间 → 竖屏改成**上下错位**，两人同框

```
9:16 竖屏，垂直错位正反打：
男主占画面上半区（特写，眼神锐利），
女主占画面下半区（特写，睫毛微颤），
均位于中间80%安全区，背景虚化
```

### 4. 主观 POV 跟拍（竖屏代入感天花板）

**功能**：模拟手机用户第一人称视角

**竖屏优势**：9:16 正好就是手机屏幕比例，观众=角色 的代入感最真实

```
9:16 竖屏，主观 POV 跟拍（模拟女主的手机视角），
手指滑动手机屏幕（特写，占中间80%安全区），
背景是晃动的地铁车厢，代入感拉满
```

### ❌ 竖屏禁用的横屏角度

| 角度 | 为什么禁用 | 负向提示词 |
|------|-----------|-----------|
| 水平摇拍全景 | 垂直画幅让摇拍生硬无意义 | `no horizontal pan` |
| 左右平移跟拍 | 主体快速移出画面 | `no truck left, no truck right` |
| 水平对称正反打 | 浪费 90% 纵向空间 | `no horizontal shot-reverse-shot` |

## 五、运镜：从"横向铺展"到"垂直流动"

### 横屏 vs 竖屏运镜对比

| 维度 | 横屏 | 竖屏 |
|------|------|------|
| **逻辑** | 横向铺展（空间广度） | 垂直聚焦（情绪强化） |
| **速度** | 2秒以上（缓慢） | **0.5-1秒（极快）** |
| **描述导向** | 环境导向 | 主体导向 + 安全区约束 |

### 基础运镜的竖屏重写

| 运镜 | 横屏方向 | 竖屏方向 |
|------|---------|---------|
| 摇镜 | 左右摇拍 | **上下摇拍**（展示全身穿搭） |
| 跟拍 | 左右跟拍 | **垂直跟拍**（奔跑楼梯间） |
| 推拉 | 水平推拉 | **垂直推拉**（推近手部特写） |

### 🔥 3 种竖屏专属复合运镜

#### ① 开场抓人三连镜（标准开场范式，3秒内完成）

```
9:16 竖屏，
开场 0.5 秒快速推近到霸总中近景定格；
立刻切到 45° 侧前方环绕半圈；
最后用微甩镜落到霸总正面特写，
保持面部在中间80%安全区，
镜头节奏"快-快-稳"
```

#### ② 垂直环绕 + 推近（情绪爆发神器）

```
9:16 竖屏，低角度垂直环绕霸总上半身
（保持面部在中间80%安全区），
同时 0.3 秒快速推近至眼部特写，
背景完全虚化，冷色调光线勾勒面部轮廓
```

#### ③ 先推后拉 + 升降（反转场景专用）

```
9:16 竖屏，
先 0.5 秒推近至女主含泪的眼睛特写；
再 1 秒后拉 + 垂直升降，
展示女主蜷缩在空荡的房间（全景），
背景虚化，仅留天花板的冷光
```

### 运镜节奏毫秒级控制

- **快速运镜**：标注 ≤0.5 秒，节奏"快-稳"（情绪爆发）
- **慢速运镜**：标注 ≥1 秒，节奏"慢-沉"（情绪铺垫）
- **复合节奏**：标注"0.3 秒快环绕 + 0.5 秒定格特写"，节奏"快-停"（反转）

## 六、转场：从"场景过渡"到"情绪延续"

### 核心认知颠覆

| 维度 | 横屏 | 竖屏 |
|------|------|------|
| **转场本质** | 场景过渡（连接物理空间） | **情绪延续**（主体动作连接） |
| **描述导向** | 场景导向（淡入淡出） | 主体导向 + 动作延续 |

> 竖屏任何生硬转场都会**打断情绪流**，导致观众跳出剧情

### 3 种竖屏专属转场

#### ① 前景擦镜转场（节奏快剧情）

```
9:16 竖屏，女主抬手捂脸（特写，占中间80%安全区）
→ 手部完全遮挡镜头时，无缝切换至雨中场景（全景），
动作延续无断裂，背景虚化
```

#### ② 道具线索转场（悬疑情感剧）

```
9:16 竖屏，女主手中的情书（特写，占中间80%安全区）
→ 镜头推近至情书文字时，无缝切换至男主的办公桌场景（中景），
情书放在桌面，视觉元素完全一致
```

#### ③ 情绪匹配转场（情感爆发）

```
9:16 竖屏，女主崩溃大哭（特写，暖黄光线打在脸上）
→ 情绪高潮时，无缝切换至男主的回忆场景（中景），
暖黄光线、背景虚化程度完全匹配
```

### 节奏参数化

| 参数 | 要求 |
|------|------|
| **节奏密度** | 每 3-5 秒一次转场或运镜切换 |
| **转场时长** | ≤0.3 秒（避免视觉空白） |
| **节奏变化** | "前3秒快（0.2秒/次）→ 后2秒慢（1秒/帧）" |

## 七、竖屏短剧 3 种热门视觉风格

| 风格 | 视觉特征 | 适配场景 | 关键词 |
|------|---------|---------|--------|
| **霸总情感风** | 冷色调/高对比度/浅景深/边缘光 | 甜宠/复仇 | 冷色调 + 人物边缘光 + 浅景深 |
| **古风逆袭风** | 暖色调/低饱和/柔光/中式对称 | 古风/穿越 | 暖色调 + 中式对称构图 + 柔光 |
| **赛博朋克风** | 霓虹光/高饱和/蓝紫/科技 UI | 科幻/悬疑 | 霓虹光影 + 蓝紫色调 + 科技感 UI |

### 参考视频学习的提示词公式

```
参考视频：[URL]，
核心风格：冷色调（权重1.2）、高对比度（权重1.1）、浅景深（权重1.0），
9:16 竖屏，低角度仰拍，主体位于中间80%安全区，
no 暖色调，no 深景深，no 杂乱背景
```

**权重规则**：
- 风格特征 > 场景描述（权重 1.2 > 1.0）
- 角色特征 > 场景描述（避免角色被场景干扰）

## 八、角色一致性的竖屏方案

### 工具能力差异

| 工具类型 | 方案 | 写法 |
|---------|------|------|
| **主流**（Seedance 2.0/Pika 3.0） | 角色 ID 锁定 | `角色ID：123，面部特征：高鼻梁、桃花眼，穿搭：黑色西装，发型：背头` |
| **旧版**（SD v1.5） | LoRA 模型训练 | `LoRA模型：[角色名称]，面部特征：高鼻梁、桃花眼...` |

### 权重分配铁律

```
角色特征（权重1.2）：高鼻梁、桃花眼、黑色西装
场景描述（权重1.0）：办公室
```

**原因**：角色特征必须高于场景，否则模型会被场景元素干扰导致"跳脸"

## 十二、核心记忆点（必须刻进工作流）

1. **提示词结构**：横屏=场景描述型，竖屏=**优先级指令型**
2. **画幅必须前置**：`9:16 竖屏` 永远放在提示词最前端（最高权重）
3. **安全区铁律**：TikTok 中间 80%（上下各 10%），抖音中间 60%（上下各 20%）
4. **景别颠倒**：竖屏开场必须是**特写/大特写**，绝不用全景
5. **专属 4 角度**：低角度仰拍 / 45°侧俯拍 / 垂直错位正反打 / 主观 POV
6. **横屏 3 禁用**：no horizontal pan / no truck left-right / no horizontal shot-reverse-shot
7. **运镜方向**：横屏左右，竖屏**上下**（摇/跟/推拉一律改垂直）
8. **运镜节奏**：毫秒级控制，0.3-0.5 秒为常态，节奏"快-稳-快-停"
9. **转场本质**：不是场景切换，是**情绪延续**（前景擦镜/道具线索/情绪匹配）
10. **权重规则**：角色特征 1.2 > 场景描述 1.0，强制模型优先保角色不跳脸
