# 闯关游戏 — 产品策划文档

> 📌 **当前版本：v1.0（策划中）**  
> 最后更新：2026-05-08  
> 本文档为头号听力闯关游戏模块的单一事实来源。MVP 先跑通 A1 难度基本闯关流程。

---

## 一、产品定位

**沉浸式第一视角情景剧 + 分支选择 + 强制/可选跟读 + 剧后微练习**，把"学英语"伪装成"玩互动剧"。

**与多邻国的本质区别**：多邻国是单题→单题的碎片化做题，闯关是一个连续的真实社交困境，每个选择都有后果，用户是在"用英语解决一件事"而不是"做英语题"。

---

## 二、关卡体系

### 2.1 三层架构

```
难度段（A1 / A2 / B1）
  └─ 主题包（每段 6~8 包，每包围绕一个生活场景）
      └─ 关卡（每包 5 关）
```

A1 总计约 25~30 关，分为 6 个主题包。主题包让"刷关卡"有叙事连续感，方便内容生产按主题批量生成。

### 2.2 主题包清单（A1 MVP）

| # | 主题包 | 场景 | 关卡数 | 解锁条件 |
|---|--------|------|--------|----------|
| P1 | 🍽️ 餐厅生存 | 点餐/结账/投诉 | 5 | 初始 |
| P2 | 🛒 购物日常 | 超市/商场/退货 | 5 | P1 通关 3 关 |
| P3 | ✈️ 机场出行 | 值机/安检/登机 | 5 | P2 通关 3 关 |
| P4 | 🏨 酒店入住 | 预订/入住/退房 | 5 | P3 通关 3 关 |
| P5 | 🚕 打车问路 | 打车/指路/约车 | 5 | P4 通关 3 关 |
| P6 | ☕ 咖啡社交 | 点咖啡/聊天/邀约 | 5 | P5 通关 3 关 |

### 2.3 单关卡剧本结构

```
[开场旁白]（3-5s）告诉用户身份+困境
   ↓
[场景 1 视频]（~8s，对方说话）
   ↓
[选择点 A] 2-3 个回复选项，用户选择后跟读
   ↓
[场景 2 视频]（根据选择走分支）
   ↓
[选择点 B]
   ↓
... （总共 3-5 个选择点）
   ↓
[结局视频]（Good / Bad）
   ↓
[结算页] 星级 + 金币 + 进入微练习
```

**每关控制在 90-120 秒剧情 + 60-90 秒微练习**，总时长 ≤ 3 分钟。

### 2.4 分支拓扑（预生成成本最优解）

采用**收束式分支**而非真·树状分支：

```
     选择点A
    ╱   │   ╲
   A1  A2  A3
    ╲   │   ╱
     选择点B  ← 三条路汇聚到同一个场景B，但对方台词会带一句回应
    ╱   │   ╲
   ...
     选择点N
    ╱       ╲
  Good      Bad  ← 仅最终结局分叉
```

每个选择点 3 个分支，分支在下一个节点收束，只有最终结局分叉。一关的视频素材数 ≈ 5(选择点) × 3(选项) + 2(结局) ≈ 17 个片段，可控。

## 三、核心交互设计

### 3.1 选项卡片

每个选项同时呈现：

```
┌────────────────────────────────────┐
│ "Excuse me, I didn't order this."  │ ← 英文台词（主）
│ "不好意思，我没有点这个。"              │ ← 中文翻译
│ 🎭 礼貌 · 直接                       │ ← 语气标签
│ [🎤 跟读后发送]  [→ 直接发送]         │ ← 双按钮
└────────────────────────────────────┘
```

**语气标签**教用户"同样意思不同说法的社交后果"：礼貌 / 强硬 / 委婉 / 幽默 / 冷淡 / 抱歉。

### 3.2 跟读机制

- **默认推荐跟读**：主按钮在"🎤 跟读后发送"
- **跟读评分 ≥70 分**：金币翻倍 + 解锁剧情加速动画
- **跟读 <70 分**：可重来一次或直接发送，不惩罚
- **跳过跟读**：剧情正常推进，但**本次不计入跟读分**（关卡评星扣一颗）
- **跟读内容来自真实选项台词**，不是单独出题——沉浸感核心

**跟读评分付费边界**：免费用户能录能听回放，但**不显示分数**，只给"✅ 已跟读"。付费用户显示分数+改进建议。

### 3.3 卡住/听不懂的兜底

每个选择点右上角：
- 🔁 **重播场景视频**（免费无限）
- 🔤 **显示字幕**（按全局字幕设置）
- 💡 **AI 教练提示**（消耗 AI 教练次数或金币，解释当前困境和每个选项的影响）

## 四、结算与评分

### 4.1 关卡评星（3 星制）

| 星 | 条件 |
|---|---|
| ⭐ | 完成剧情（任何结局） |
| ⭐⭐ | 达成 Good Ending |
| ⭐⭐⭐ | Good Ending + 所有选择点都完成跟读（≥70 分） |

3 星解锁"大师回放"（隐藏字幕重看一遍，免体力）。

### 4.2 结算页内容

- 结局标识（Good / Bad）
- 星级 + 金币奖励（基础 + 跟读翻倍）
- 本关学到的核心短语（自动进入收藏夹"闯关解锁"分组）
- 「继续练习」→ 进入微练习
- 「重新挑战」→ 再玩一次

### 4.3 微练习衔接

微练习直接复用本关视频做题源，优先用：
- 选台词含义（用分支视频）
- 预测完型（挖空用户选过的句子）
- 8 词选 4（从本关所有对话中选）

## 五、体力系统

### 5.1 基础规则

| 项 | 数值 | 说明 |
|---|---|---|
| 体力上限 | 免费 5 / 付费 8 | 新用户首日 10 满格 |
| 恢复速度 | 30 分钟 / 1 点 | 免费；付费 20 分钟/点 |
| 单关消耗 | 1 点 | 进入即扣，无论成败 |
| 重玩消耗 | 1 点 | 同关重玩也扣 |
| 大师回放 | 0 体力 | 三星关专属奖励 |

### 5.2 体力来源（5 条路径）

1. **自然恢复**：30 分钟 +1（基础）
2. **每日签到**：连续签到额外送 +2（不破上限）
3. **完成任务**：今日任务送 +1~+2
4. **金币兑换**：50 💰 = 1 体力，免费上限日 3 次
5. **看广告**：15 秒广告 +1，上限日 3 次（可选）

### 5.3 体力不足弹窗

引导用户：
- 金币兑换体力（50💰/点）
- 看广告补体力
- 升级会员（永不断电）
- 或回到 APP 其他功能（刷短视频、复习收藏词）

## 六、金币系统

### 6.1 金币流向

```
       【产出】                    【消耗】
  ┌─────────────┐           ┌─────────────┐
  │ 通关  +20    │           │ 买体力 -50   │← 核心回路
  │ 三星 +30    │           │ AI提示 -10   │
  │ 跟读 +10    │  ═══════> │ 复活 -30     │
  │ 日常 +50    │           │ 会员体验 -500 │
  │ 签到 +30    │           │ 皮肤 -300~800 │
  └─────────────┘           └─────────────┘
```

### 6.2 经济平衡

**F2P 用户一天正常产出约 210 💰**（5 次体力 × 20 + 日常 80 + 签到 30）  
**一天正常消耗约 70~120 💰**（换 1~2 次体力 + AI 提示）  
**日均盈余约 90 💰**，积攒 6~7 天可买大额消费品。

### 6.3 消耗点优先级

| 消耗点 | 定价 | 目的 | V1 |
|---|---|---|---|
| 买体力 | 50/点 | 核心循环 | ✅ |
| AI 教练提示 | 10/次 | 关卡内救急 | ✅ |
| 复活（失败不扣体力） | 30 | 降低挫败 | ✅ |
| 会员体验券（3 天全功能） | 500 | 免费→付费桥 | ✅ |
| 关卡皮肤/主题 | 300-800 | 金币大额出口 | ⏳ V1.2 |
| 学习报告/周报 | 100 | 付费功能试吃 | ⏳ V1.1 |

## 七、每日/周任务

### 7.1 每日任务（每日 04:00 刷新，随机抽 3 个）

| 任务 | 奖励 |
|---|---|
| 通关 3 关 | +50 💰 |
| 三星通关 1 关 | +100 💰 |
| 跟读 10 句 ≥70 分 | +30 💰 |
| 刷短视频 5 条 | +20 💰 |
| 复习 5 个即将降级词 | +30 💰 |
| 完成 1 次 AI 教练对话 | +20 💰 |
| 挑战同难度新主题包 | +50 💰 |

**设计原则**：任务必须同时喂养闯关和其他功能，避免用户只会玩闯关。

### 7.2 周任务（周一刷新）

| 任务 | 奖励 |
|---|---|
| 周通关 20 关 | 会员体验券（3 天）|
| 周跟读 50 句 | 跟读评分解锁 1 次 |
| 周连续登录 7 天 | 50 💰 + 5 体力 |

## 八、入口页设计

### 8.1 页面结构（从上到下）

1. **顶部 HUD**：体力（含恢复倒计时）+ 金币 + 连续天数
2. **当前关卡 Hero 卡**：场景画面 + 关卡名 + 上次成绩 + 「开始挑战」CTA
3. **难度切换 Tab**：A1 基础 / A2 进阶 / B1 挑战
4. **蛇形路径**：主题包 × N，关卡节点带状态区分
5. **今日任务**：3 个任务 + 进度
6. **签到墙**：7 天签到格子
7. **底部导航**：短视频 / 闯关 / 收藏 / 我的

### 8.2 关卡节点状态

| 状态 | 视觉 | 行为 |
|---|---|---|
| 🔒 未解锁 | 灰色 | 不可进入 |
| 🟠 当前关 | 橙色 glow 闪烁 | 消耗 1 体力进入 |
| ⭐ 已通关未满星 | 金边 | 可重挑战 |
| ⭐⭐⭐ 三星 | 三金星 | 可大师回放（免体力）|
| 👑 主题包 Boss 关 | 皇冠图标 | 消耗 2 体力，难度稍高 |

### 8.3 新用户首日保护

新用户第一天给 **10 体力满格**（不是 5），让他们一口气玩爽——第二天恢复正常上限。防止首日体验被打断导致流失。

## 九、与现有体系打通

| 现有体系 | 闯关游戏如何打通 |
|---|---|
| 收藏夹 | 关卡核心短语自动入"闯关解锁"分组 |
| 熟悉度 | 关卡内收藏词做对选择 = 答对一题，参与升级 |
| 重遇弹幕 | 刷短视频时，闯关学过的词触发弹幕（文案："上周《餐厅对账》学到"）|
| AI 教练 | 关卡内 AI 提示 = 消耗教练次数或金币 |
| 跟读评分 | 完全复用现有评分引擎 |
| 金币 | 统一金币池，闯关是最大产出源 |
| 字典 | 选项卡片长按单词→字典 |

**设计原则**：闯关是"主动学习入口"，短视频流是"被动沉浸入口"，两者喂养同一个收藏夹 → 熟悉度系统 → 复习系统。

## 十、难度差异

| | A1 | A2 | B1 |
|---|---|---|---|
| 场景复杂度 | 单次对话 | 1-2 个转折 | 多方+情绪张力 |
| 单句长度 | ≤8 词 | ≤15 词 | ≤25 词 |
| 语速 | 0.8× | 0.9× | 1.0× |
| 字幕默认 | 中英 | 减压 | 纯英 |
| 选项差异 | 明显对错 | 语气差异 | 策略差异（都对但后果不同）|
| AI 提示 | 给中文意思 | 给关键词 | 只给情境提示 |
| 每关选项数 | 2 选 1 | 3 选 1 | 3 选 1 + 结局 3 种 |

## 十一、付费边界

| 项 | 🆓 免费 | 👑 付费 |
|---|---|---|
| 难度解锁 | A1 全部开放 | A2 / B1 全部 |
| A2 以上 | 每主题包首关体验 | 全部关卡 |
| 体力上限 | 5 | **8** |
| 体力恢复 | 30 分/点 | **20 分/点** |
| 每日金币换体力 | 3 次 | **无限** |
| 看广告补体力 | ✅ | 不用看 |
| 大师回放 | 仅三星关 | **所有已通关** |
| 跟读评分 | ❌（仅显示"已跟读"）| ✅ 评分+改进建议 |
| 闯关内 AI 提示 | 消耗金币 | 每关免费 1 次 |
| 关卡难度筛选 | ❌ 顺序解锁 | ✅ 跳关挑战 |

**不做**：
- ❌ 关卡单独收费（按关买解锁是手游逻辑，破坏年付生态）
- ❌ 体力值（除体力系统外不加限制层，已有金币和每日额度）

## 十三、生产端注意点

- **视频素材**：每关 ~17 个视频片段，需工业化 AIGC 生成（Sora/Runway/Veo）
- **剧本格式**：建议 JSON 结构化剧本，方便内容团队批量生产
- **分支容错**：用户连续选"差"选项也应该能走到某个结局，不能卡死
- **首日体验**：新用户第一关应该是"精选体验关"，剧本写得特别精彩，确保留下好印象
